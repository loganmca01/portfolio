Customizable Toy Assembly Interpreter (in progress)
GitHub Repo

• Toy assembly interpreter where the user can enter an instruction set using a custom, easy-to-read register transfer notation, then run the commands dictated in the instruction set in a virtual environment.
• Designed for use in Computer Organization class to introduce assembly concepts without full assembly syntax.
• Two main components:

1. Instruction set parser: Takes a file input of instructions written in register transfer notation. Builds abstract syntax trees for each command, then puts them into symbol table for use in the user environment.
2. User command interpreter: Takes user commands (file or stdin) and runs them in virtual environment with an array of chars representing the stack, and variables for registers.

Instruction set example:

define ret {
    SP := BP;
    BP := (SP);
    SP := SP + 2;
    PC := (SP);
}

LAB 4:
• Focused around building fetch-decode-execute cycle for a toy assembly calculator. No string.h or built in string functions were allowed, we were required to build any string parsing or comparison functions ourselves. Registers were represented as binary strings. Addition, subtraction, bitwise & |, negation, shifts, and conversions to decimal and hexadecimal were all implemented going character by character through strings. No stack, just registers in simulated environment.

LAB 5:
• Focused on expanding virtual environment from lab 4 with a more complex assembly language. Created "stack" with array of memory contents structs, which were built to store all relevant information about an item in memory, such as the command and arguments if the memory held an instruction.
